【ソフト紹介】
  「かぞえチャオ！」は、ソースコードのステップ数をカウントするツールです。
  ステップ数のカウントのほか、変更前と変更後のソースコードのモジュール比
  較を行い、新規、修正元、修正、流用ステップ数の算出もできます。
  ソフト開発における、生産性を評価するときに使えるツールです。
  主な特徴は、以下のとおりです。

  ・拡張子個別情報ファイルを追加することにより、さまざまな言語で記述され
    たプログラムのステップ数カウントに対応できます。
  ・C、C++、VB、Java、MASM用ファイルを、標準セットで用意しています。
  ・ステップ数のカウントは、ファイル単位のほか、モジュール単位、クラス単
    位でもカウントできます。
    ステップ数のカウント結果は、別ウィンドウに一覧表示されます。
  ・HTML中にあるスクリプト(Java, VB etc)のモジュール切り出し、ステップカ
    ウントも、拡張子個別情報ファイルのあわせ技でできます。
  ・ファイルの指定は、フォルダを指定するだけです。
    エクスプローラからのドラッグ＆ドロップでもファイルが指定できます。
    フォルダを指定すれば、チェック対象ファイルだけをリストアップして一覧
    表示します。
  ・リストアップされたファイルのうち、チェック不要なファイルは後で削除も
    できます。
  ・変更後と変更前のフォルダを選択して、ファイルのステップ比較もできます。
    結果は、ステップ数カウント結果と同様、別ウィンドウで一覧表示します。
    新規、修正元、修正、流用ステップ数がカウントできます。
  ・行区切りとして、CR+LF、CR、LF、AUTOが選択できます。
    AUTOを選択すると、行区切りを自動認識します。
  ・カウント結果は、CSV、HTML、TEXT形式のファイルとして保存できます。
    Excelで結果が編集できます。
    Webブラウザで、表として参照することもできます。
  ・カウント結果をタグファイルとしても保存できます。
  ・結果一覧画面では、各項目のソートもできますので、ステップ数の多い順、
    修正量の多い順に並べ替えて参照が可能です。
  ・Windows 7,Vista,XPで動作します。
    VBランタイム、DLLファイル、OCXファイルは不要です。
  ・ファイル編集呼び出し機能により、結果一覧から直接エディタを起動するこ
    ともできます。
  ・結果プロパティでは、クラス数、モジュール数も確認できます。
  ・Ver.1.12以降では、起動時に自動的にファイルを読み込んで、ステップ比較、
    ステップカウントを行うこともできます。結果を、指定フォルダに保存して、
    自動終了することもできます。

    Ver.1.12β2
    - プリプロセッサ(#ifdef、#if など)を含むソースコードからのモジュール
      切り出し、ステップカウントに対応

      拡張子個別情報ファイルに下記を記述することで、#if や #ifdef 以降の
      コードを有効にしたり、無効にすることができます。

      [PREPRO]
      種類（最大64種類まで）
      １種類目の終了文字列の数, 開始文字列見つかった後(0:有効 1:無効)
      開始文字列
      終了文字列１
          ：
      終了文字列Ｎ（Ｎ＝最大８まで）
      ２種類目の終了文字列の数, 開始文字列見つかった後(0:有効 1:無効)
      開始文字列
      終了文字列１
          ：
      終了文字列Ｎ（Ｎ＝最大８まで）

      ※開始文字列の先頭部分が同じ文字列を登録する場合は、文字列長の長いもの
        から順に記述してください。
        タブや空白を含む文字列は、半角スペース１個で文字列を区切ってください。

     <設定例>
      [PREPRO]
      3
      2,1
      #ifdef
      #else
      #endif
      2,0
      #ifndef
      #else
      #endif
      2,1
      #if
      #else
      #endif

      上記の設定例では、

      #ifdef AAA
          a = 1;        →無効
      #else
          a = 2;        →有効
      #endif
      #ifndef BBB
          b = 3;        →有効
      #else
          b = 4;        →無効
      #endif
      #if 0
          c = 5;        →無効
      #else
          c = 6;        →有効
      #endif

      のようになります。
      なお、プリプロセッサ行は、実ステップ数として数えません。

      #ifdef AAA だけを有効にする場合は、

      [PREPRO]
      5
      2,0
      #ifdef AAA
      #else
      #endif
      2,1
      #ifdef
      #else
      #endif
      2,1
      #ifndef AAA
      #else
      #endif
      2,0
      #ifndef
      #else
      #endif
      2,1
      #if
      #else
      #endif

      とすればＯＫです。

    Ver.1.12β6
    - 自動実行機能を追加
      システム設定メニューに、自動実行ファイルの指定を追加しました。
      自動実行ファイルの基本拡張子は、「.cas」です。
      指定した自動実行ファイルがあれば、起動時に読み込んで自動実行します。

    ・自動実行ファイルの記述内容について
      以下の記述により、変更後ファイル、変更前ファイルをリストに登録し、
      ステップ比較を行います。

      [AFTPATH]
      C:\usr\bctest\kazoe112\testsrc
      [BFRPATH]
      C:\usr\bctest\kazoe111\testsrc
      [EXEMODE]
      1

      さらに、以下を追加すると、結果を指定フォルダに保存して自動終了する
      ことができます。

      [RSLMODE]
      0
      [RSLPATH]
      C:\usr\steprslt

      ------------------------------------------------------------------
      [AFTPATH]     変更後リストに登録するフォルダ、ファイルを指定
                    複数のフォルダ、ファイルを連続で指定できます。
      ------------------------------------------------------------------
      [BFRPATH]     変更前リストに登録するフォルダ、ファイルを指定
                    複数のフォルダ、ファイルを連続で指定できます。
      ------------------------------------------------------------------
      [EXEMODE]     実行種別の数値を指定
                    1 ステップ比較を実行
                    2 変更後ステップカウントを実行
                    3 変更前ステップカウントを実行
                    これ以外の数値は無効。実行しません。
      ------------------------------------------------------------------
      [RSLMODE]     実行結果保存ファイル種別の数値を指定
                    0 CSV
                    1 HTML
                    2 TXT
                    これ以外の数値は無効。結果保存、自動終了しません。
      ------------------------------------------------------------------
      [RSLPATH]     実行結果を保存するフォルダを１つ指定
                    すでに存在するフォルダを指定してください。
                    フォルダがない場合は、結果保存、自動終了しません。
      ------------------------------------------------------------------

      ファイルがなくて、実行できなかった場合は、自動終了しません。

        0:開始文字列発見後の振る舞い(0:コード無効にしない 1:コード無効にする)
        1:開始文字列にマクロを付加(0:しない 1:する)
        0:終了文字列１にマクロを付加(0:しない 1:する)
        0:終了文字列２にマクロを付加(0:しない 1:する)

        #ifdefには、マクロを付加する
        #else には、マクロを付加しない
        #endifには、マクロを付加しない

    ※無効なマクロは、マクロオプションに記述する必要ありません。
      上記の例では、ABCDとEFGHのマクロのみ有効になります。

    ＜具体例＞

      #ifdef ABCD             ←このマクロは登録されている
          ここのコードは有効
      #else
          ここのコードは無効
      #endif
      #ifdef DEBUG            ←このマクロは登録されていない
          ここのコードは無効
      #else
          ここのコードは有効
      #endif
      #ifndef EFGH            ←このマクロは登録されている
          ここのコードは無効
      #else
          ここのコードは有効
      #endif
      #if 0                   ←このマクロは登録されていない
          ここのコードは無効
      #endif
      #if ABCD                ←このマクロは登録されている
          ここのコードは有効
      #endif

    Ver.1.53β5
    - プリプロセッサのマクロ演算に対応しました。
      マクロ演算できるのは、

      defined()   マクロ定義確認
      &&          AND演算
      ||          OR演算
      !           NOT演算
      +           加算
      -           減算
      ==          同値
      <=          以下
      >=          以上
      <           より小
      >           より大

      です。掛け算と割り算には対応してません。
      defined() を、&&や||で結合した場合でも、正しく有効、無効を判断できる
      ようになりました。
      なお、行末の\ による行接続には対応していません。

      この機能を使用するためには、拡張子個別情報ファイルの[PREPRO]セクショ
      ンを変更しなければいけません。

      [PREPRO]
      5
      2,0,1,0,0
      #ifdef
      #else
      #endif
      2,1,1,0,0
      #ifndef
      #else
      #endif
      3,0,2,2,0,0    ←この行を左記のように変更
      #if
      #elif
      #else
      #endif
      1,1,0,0
      #else
      #endif
      0,1,0
      #endif

      #if defined、#if !defined に関する記述があれば削除します。
      #if に関するデータを、以下のように変更します。

        旧(従来)        新(今回)
        3,0,1,1,0,0     3,0,2,2,0,0
        #if             #if
        #elif           #elif
        #else           #else
        #endif          #endif

      数値データの３つ目と４つ目のパラメータを１→２に変更しています。
      ３つ目と４つ目のデータは、#if および#elif へのマクロ定義の付加指定を
      示しています。
      パラメータの意味は、以下のとおりです。

      ０＝マクロ定義をプリプロセッサに付加しない
      １＝マクロ定義をそのままプリプロセッサに付加して、そのままコードと比
          較
      ２＝コードのプリプロセッサのマクロ定義部の演算をして、有効無効の判断
          をする

    - マクロ定義に数値データを追加できるようにしました。

      #define AAA 10
      というマクロをプリプロセッサ(#if)で利用している場合、

      [MACRO]
      1
      AAA,10

      のように拡張子個別情報ファイルの[MACRO]セクションに定義することで、

      #if AAA == 10
          :
      #endif

      を有効コードとすることができるようになりました。
      プリプロセッサのマクロ演算処理と組み合わせることで、この処理を行って
      います。

      た。（以前は、半角4文字でした。）

        CGIのヒアドキュメントのみの文字列認識は、下記のように記述します。

        [STRING]
        1

        %s
        None

        ※任意の文字列の指定を「%s」で行います。

               :
            ABC
          の間をすべて文字列と認識します。

          %s部分は、0〜9(0x30〜0x39)とA〜(0x41〜)の文字コードのみ有効です。

          << "ABC";
          とした場合でも、文字列終了文字列は、ABCとなります。
